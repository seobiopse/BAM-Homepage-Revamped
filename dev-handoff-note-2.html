What this document covers

Overview

This hand-off note details the **static compilation framework and template architecture** built for badgestore.com.au (BAS). To ensure lightning-fast loading speeds, complete offline preview compatibility, and robust search indexing, the site compiled to flat, standalone HTML files without database or dynamic backend requirements.

The architecture uses an Isomorphic Catalog Datastore (`assets/data.js`) defining categories and products, a local Node.js Compiler (`build.js`), and three semantic **HTML Layout Templates** (`templates/`).
1
Build Environment

Compilation Workflow

The compiler compiles all page-level categories, sub-categories, and products dynamically into static files. Relative link paths are rewritten recursively to ensure standard subfolder compatibility (e.g. on GitHub Pages).

Input Files (Source Templates)
Isomorphic datastore:
BAS/assets/data.js (Products, pricing brackets, categories, meta configurations)

HTML templates:
  • `templates/index-template.html`
  • `templates/category-template.html`
  • `templates/product-template.html`
Output Structure (Compiled Production Files)
Production Root:
BAS/index.html (Homepage)

Subdirectories:
  • `categories/[category-slug].html` (e.g., `acrylic-dome-name-badges.html`)
  • `products/[product-slug].html` (e.g., `acrylic-domed-name-badge.html`)
# To recompile the entire website static HTML structure, run:
node build.js

# Light-weight local preview server command:
npx -y http-server . -p 8080
2
Data Model

Isomorphic Catalog Registry (`assets/data.js`)

All website parameters are defined in a clean isomorphic JSON registry module. This object is imported by `build.js` in Node, and loaded directly by client-side configurator scripts in the browser.

Registry Field Data Type Purpose & ACF Equivalent
categories[slug].parent String | null Identifies if a category is a sub-category. Influences dropdown nesting and slug routing.
categories[slug].metaTitle String Page title tag parsed by crawler spiders and search engines.
categories[slug].metaDesc String SEO snippet description shown on search engine result pages.
categories[slug].comparisonGuide Object Holds headers and cell rows for rendering dynamic comparison matrices.
categories[slug].faqs Array<Object> List of question/answer blocks mapping to responsive FAQ Accordions and JSON-LD FAQ schema.
products[slug].priceBrackets Array<Object> Wholesale bulk purchase tiers used by client-side calculators.
💡
Isomorphic Compatibility Hook To run correctly in both server-side Node execution environments and client-side web browser clients, the catalog file ends with: 
if (typeof module !== 'undefined' && module.exports) {
    module.exports = ProductCatalog;
} else {
    window.ProductCatalog = ProductCatalog;
}
3
Placeholders mapping

Template Placeholder Interpolation

Double braces syntax {{variable}} represents content insertion areas inside raw templates files. The builder script parses templates, replaces these strings, and saves output HTML.

Placeholder Tag Scope Template Generated Content Source
{{metaTitle}} Category, Product Interpolated from registry metadata configs.
{{metaDesc}} Category, Product Interpolated from registry meta descriptions.
{{navMenu}} All templates Returns nested navigation structures. Evaluates parent-child category linkages.
{{breadcrumbs}} Category, Product Renders hierarchical navigation trail indicating current position.
{{seoArticle}} Category Renders the redesigned, structured, feature-rich editorial copy layout.
{{comparisonTable}} Category Draws standard responsive comparison guides.
{{faqList}} Category, Product Renders accessible accordion tabs with toggle micro-animations.
{{schemaGraph}} All templates Outputs programmatic SEO structured metadata.
4
Layout Redesign

SEO Description Redesign Extractor

Rather than displaying messy unstyled raw HTML blocks scraped from legacy source files, build.js parses old category content via regular expressions and generates standard responsive sections.

Original Content Elements
Messy raw assets:
Legacy category descriptions contain raw inline CSS, tables, and images placed in a single continuous body column, leading to layout squeeze.
Redesigned Components Output
1. Hero Feature Block: Image on the left with shadow, title/description and wholesale price alerts on the right.
2. Color Swatch Grid: Inline swatches display inside a rounded chip list with hover tooltips.
3. Backing Clip Cards: Renders a responsive card grid mapping standard clips to photos and details.
4. Editorial Content: Left-aligned text headings with clean margins.
⚡
Critical Design & Styling Alignments
  • Standalone Logo: Logo height is set strictly to 42px with margin-right: 40px in header styles.
  • Single-Line CTA: Header CTA button has white-space: nowrap !important and tensed horizontal padding. Keeps "DIY Badges" CTA aligned on one single line.
  • Cache Buster: Stylesheet links must always use cache buster parameters (e.g. style.css?v=2) to prevent browser rendering discrepancies.
5
Generative Search

Programmatic JSON-LD Schema Graph

To achieve maximum Generative Engine Optimization (GEO) and Answer Engine Optimization (AEO), the static compilation script automatically injects dynamic structured metadata.

// Generated programmatically per category file inside build.js:
const catSchema = {
    "@context": "https://schema.org",
    "@graph": [
        {
            "@type": "CollectionPage",
            "@id": `https://www.badgestore.com.au/categories/${cat.slug}.html#collection`,
            "name": cat.heading,
            "description": cat.metaDesc,
            "url": `https://www.badgestore.com.au/categories/${cat.slug}.html`
        },
        {
            "@type": "ItemList",
            "itemListElement": cat.products.map((pRef, idx) => ({
                "@type": "ListItem",
                "position": idx + 1,
                "url": `https://www.badgestore.com.au/products/${pRef.id}.html`,
                "name": pRef.name
            }))
        }
    ]
};
