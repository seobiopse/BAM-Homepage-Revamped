# BadgeStore Australia - Programmatic Multi-Page Static Site Redesign

This repository contains the premium, mobile-optimized redesign of the [BadgeStore](https://www.badgestore.com.au/) e-commerce catalog site. 

The project is structured as a **statically compiled multi-page website** built using pure HTML, Vanilla CSS, and Vanilla JavaScript, ensuring zero third-party framework dependencies, lightning-fast performance, and absolute compatibility with **GitHub Pages** static site hosting.

---

## 📂 Repository Structure

The folder structure is organized cleanly to separate raw templates, reference materials, core assets, and compile-ready public files:

```
├── assets/                    # Core static assets shared across pages
│   ├── style.css              # Unified design system & responsive layout styles
│   ├── app.js                 # Shared interactivity logic (Previewers, Calculators)
│   └── data.js                # Centralized catalog datastore registry (Isomorphic JS)
├── templates/                 # Raw HTML template layout shells used by compiler
│   ├── category-template.html # Template shell for collection pages
│   └── product-template.html  # Template shell for product detail pages
├── references/                # Reference material, instructions, and mockups (ignored in production)
├── build.js                   # Node.js compilation builder script
├── index.html                 # Main website homepage (production file)
├── README.md                  # Project documentation and deployment guide
└── *.html                     # Compiled production HTML files (generated by build.js)
```

---

## ⚡ How it Works (Dynamic Static Site Compilation)

To provide an optimal developer experience and allow SEO crawlers to index **every unique product and category page** under clean, static URLs, we use a custom builder script:

1. **Centralized Data Layer (`assets/data.js`):** All details (prices, wholesale brackets, specifications, metadata, breadcrumbs, and FAQs) are defined programmatically in a structured JSON catalog database.
2. **Templates (`templates/`):** Define the semantic HTML structures, accessibility ARIA tag systems, and styling hooks.
3. **Local Compiler (`build.js`):** Parses the registry, interpolates the templates, generates custom JSON-LD schemas, and writes individual static HTML pages directly to the root folder (e.g. `name-badges.html`, `premium-acrylic-desk-plate.html`).

## 🛠️ Local Development & Build Commands

### 1. Compile Static Pages
If you make changes to the product catalog details in `assets/data.js` or the design structures in `templates/`, execute the build script to compile and update all physical HTML pages:
```bash
node build.js
```

### 2. Local Testing (Running a Server)
Because this is a static site, you can host and preview it locally using any lightweight HTTP server. For example, using Node's `http-server`:
```bash
# Install and start a local server at port 8080
npx -y http-server . -p 8080
```
Open `http://localhost:8080` in your web browser.

## 🚀 Deployment to GitHub Pages

This repository is optimized for direct static publishing to GitHub Pages:

1. **Commit & Push:** Push all compiled HTML files (in the root directory) and the `assets/` directory to your GitHub repository's main branch.
2. **Configure Settings:**
   * Go to your repository settings page on GitHub.
   * Under the **Pages** section on the left sidebar.
   * Set the **Source** to "Deploy from a branch".
   * Select your branch (e.g. `main`) and root folder (`/root`), then click **Save**.
3. **Link Integrity:** All links are relative (`./`) rather than absolute (`/`), meaning the site will render perfectly even when hosted under GitHub Pages subdirectories (e.g., `https://<username>.github.io/<repository-name>/`).
